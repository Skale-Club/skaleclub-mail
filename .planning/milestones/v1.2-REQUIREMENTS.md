# Requirements — v1.2 Mail Server Production

**Milestone goal:** A user with a registered, verified domain can configure `user@skale.club` in Thunderbird and send/receive email to/from the public internet, with outbound DKIM-signed and inbound SPF/DMARC-verified.

---

## Requirements

### TLS (mail ports)

| ID | Requirement | Verified by |
|---|---|---|
| TLS-01 | Let's Encrypt cert issued for `mail.skale.club` and persisted on Hetzner host | `ls /etc/letsencrypt/live/mail.skale.club/fullchain.pem` returns a file |
| TLS-02 | Docker container reads cert at startup via `MAIL_TLS_CERT_PATH`/`MAIL_TLS_KEY_PATH` env vars | Server log on boot: `[MAIL-TLS] TLS certificate loaded`; IMAP log: `SSL/TLS (implicit)` |
| TLS-03 | Certificate auto-renews without manual intervention | `systemctl status certbot.timer` active; post-renewal hook reloads container |

### DNS / Autodiscovery

| ID | Requirement | Verified by |
|---|---|---|
| DNS-01 | All required DNS records published and verified | `POST /api/domains/:id/verify` returns `verified` for spf/dkim/dmarc/mx/returnPath |
| DNS-02 | `autoconfig.skale.club` CNAME → `mail.skale.club` | `dig autoconfig.skale.club CNAME` returns the CNAME |
| DISCO-01 | Thunderbird autoconfig GET returns valid XML | `curl http://autoconfig.skale.club/.well-known/autoconfig/mail/config-v1.1.xml?emailaddress=user@skale.club` returns `<clientConfig>` with correct host/port/security |

### DKIM / Email Authentication

| ID | Requirement | Verified by |
|---|---|---|
| DKIM-01 | Outbound mail is DKIM-signed using `dkim_private_key` from the sender's domain row | Mail-tester.com score ≥ 9/10; Gmail "show original" shows DKIM=pass |
| AUTH-01 | Inbound MX receiver adds `Authentication-Results` header with SPF/DKIM/DMARC results | `mx-server.ts` logs include verification; received message has the header |
| AUTH-02 | Emails with `dmarc=fail + policy=reject` are rejected at SMTP (550) | Spoofing test from external domain is refused with correct SMTP response |

### Operations / MX Hardening

| ID | Requirement | Verified by |
|---|---|---|
| OPS-01 | Hetzner port 25 unblocked via support ticket | `nc -vz mail.skale.club 25` from external network succeeds |
| MX-01 | Per-IP rate limit + DNSBL check active in `onConnect` | Spam IP (Spamhaus-listed) is rejected with `554 5.7.1 Blacklisted` |
| MX-02 | Greylisting implemented for new (IP, from, to) triples | First attempt gets `451`, retry after ≥5min succeeds |

---

## Success Definition

End-to-end test: a Gmail user sends an email to `vanildo@skale.club`. Within 30 seconds, the message appears in the user's Thunderbird INBOX (via IMAP `IDLE`). The user replies; Gmail receives the reply with DKIM=pass, SPF=pass, DMARC=pass.

This passes when all 10 requirements above are verified.
