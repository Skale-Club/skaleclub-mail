# Phase 17 Plan 01: Logger Foundation Summary

**One-liner:** Structured pino logger (`src/server/lib/logger.ts`) + outreach alert threshold constants, replacing ad-hoc `console.log` for all Phase 17-02/03/04 downstream callers.

## What was built

A single 94-line module that downstream plans import from:

```typescript
import {
    logger,
    createLogger,
    OUTREACH_BOUNCE_WARN_1H,
    OUTREACH_BOUNCE_ERROR_24H,
    OUTREACH_PROCESSOR_SLOW_MS,
} from '../lib/logger'

// Module-scoped child logger with bound `module: 'outreach.processor'` field
const log = createLogger('outreach.processor')

log.info({ action: 'outreach.processor.tick.start' }, 'tick start')
log.warn({ action: 'outreach.bounce.detected', bounceRate, threshold: OUTREACH_BOUNCE_WARN_1H }, 'bounce 1h threshold exceeded')
```

## Logger Configuration

### Dev mode (`NODE_ENV !== 'production'`)

- Transport: `pino-pretty` (colorized output, `SYS:standard` translateTime, ignore pid/hostname noise)
- Default level: `debug` (overridable via `LOG_LEVEL`)
- Purpose: human-readable single-line output during `npm run dev:server`

### Prod mode (`NODE_ENV === 'production'`)

- Transport: none — default JSON serializer to stdout (captured by Docker → Hetzner host logs)
- `base`: `{ pid, hostname: process.env.HOSTNAME }` — disambiguates multi-replica logs
- `timestamp`: `pino.stdTimeFunctions.isoTime` (ISO-8601 strings, greppable)
- `formatters.level`: returns `{ level: label }` — emits string level names instead of numeric (matches the shape downstream `jq` queries assume per 17-CONTEXT.md)
- Default level: `info` (overridable via `LOG_LEVEL`)
- Purpose: `docker logs skaleclub-mail 2>&1 | jq 'select(.action=="outreach.send.failed")'`

## Exported Threshold Constants

| Constant                       | Value     | Meaning                                                                                                              |
| ------------------------------ | --------- | -------------------------------------------------------------------------------------------------------------------- |
| `OUTREACH_BOUNCE_WARN_1H`      | `0.05`    | Rolling-1h bounce rate above which the processor logs WARN; first signal of campaign rejection at scale              |
| `OUTREACH_BOUNCE_ERROR_24H`    | `0.10`    | Rolling-24h bounce rate above which processor logs ERROR; the deliverability cliff (Gmail/Yahoo sandboxing risk)     |
| `OUTREACH_PROCESSOR_SLOW_MS`   | `30000`   | Tick duration above which we emit `outreach.processor.tick.slow` WARN; matches the 30s cron cadence floor            |

Both rate constants are decimals (0.05 = 5%), not percentages, so callers do raw `if (bounceRate > OUTREACH_BOUNCE_WARN_1H)` math without divide-by-100.
