<objective>
Add column filtering to all list endpoints to exclude large TEXT/JSONB columns (htmlBody, plainBody, attachments, headers, spamChecks)

Purpose: List endpoints currently SELECT all columns including multi-MB body content that users never see in list views. Filtering these reduces payload sizes dramatically and speeds up queries.

Output: Modified list endpoint handlers in messages, mail/messages, campaigns, and templates routes using Drizzle `columns` filter.
</objective>

Drizzle `columns` filter syntax:
```typescript
db.query.table.findMany({
    columns: {
        htmlBody: false,
        plainBody: false,
        attachments: false,
        headers: false,
        spamChecks: false,
    },
    // ... other options
})
```

For nested relations (campaigns → sequences → steps):
```typescript
with: {
    sequences: {
        with: {
            steps: {
                columns: {
                    htmlBody: false,
                    plainBody: false,
                    htmlBodyB: false,
                    plainBodyB: false,
                },
            },
        },
    },
},
```
</context>
