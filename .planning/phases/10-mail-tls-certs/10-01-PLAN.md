# Phase 10 Plan 01 — Install certbot, mount certs, wire deploy

## Preconditions
- Phase 11 (DNS) MUST be complete first: `mail.skale.club` A record must resolve to the Hetzner VPS before Let's Encrypt HTTP-01 can succeed
- SSH access to Hetzner as root

## Tasks

### 1. One-time host setup (manual, runs once on Hetzner)
```bash
ssh root@<HETZNER_HOST>

# Install certbot
apt update && apt install -y certbot

# Stop Caddy briefly (needs port 80)
systemctl stop caddy

# Issue cert (HTTP-01 challenge via standalone server on :80)
certbot certonly --standalone \
  --non-interactive --agree-tos \
  --email ops@skale.club \
  -d mail.skale.club

# Restart Caddy
systemctl start caddy

# Verify cert files exist
ls -la /etc/letsencrypt/live/mail.skale.club/
# Expected: fullchain.pem, privkey.pem, cert.pem, chain.pem (symlinks)

# Install post-renewal hook (reload container after renewal)
cat > /etc/letsencrypt/renewal-hooks/deploy/skaleclub-mail.sh <<'EOF'
#!/bin/bash
docker restart skaleclub-mail
EOF
chmod +x /etc/letsencrypt/renewal-hooks/deploy/skaleclub-mail.sh

# Test renewal dry-run
certbot renew --dry-run
```

### 2. Update `.github/workflows/deploy-hetzner.yml`
Add to both `docker run` invocations (primary at line ~113 and rollback at line ~206):

  $IMAGE_NAME:latest

### 6. Deploy + verify
```bash
# From local
git push origin main

# On Hetzner (after deploy finishes)
docker logs skaleclub-mail | grep -i tls
# Expected:
# [MAIL-TLS] TLS certificate loaded
# [SMTP] Submission server listening on port 587 — plaintext + STARTTLS
# [IMAP] Server listening on port 993 — SSL/TLS (implicit)
# [MX] Inbound MX receiver listening on port 25 — plaintext + STARTTLS

# From external
openssl s_client -connect mail.skale.club:993 -servername mail.skale.club -brief
# Expected: chain validates, subject=mail.skale.club, issuer=Let's Encrypt
```

## Verification checklist (TLS-01, TLS-02, TLS-03)
- [ ] `/etc/letsencrypt/live/mail.skale.club/fullchain.pem` exists on host
- [ ] Container startup log shows "TLS certificate loaded"
- [ ] IMAP log shows "SSL/TLS (implicit)"
- [ ] `openssl s_client -connect mail.skale.club:993` shows valid cert
- [ ] `certbot renew --dry-run` succeeds
- [ ] Renewal hook triggers `docker restart skaleclub-mail`
- [ ] Thunderbird IMAP connect succeeds with SSL/TLS, no warning

## Risk
- Caddy downtime during initial `certbot --standalone` run (~30s) — schedule in low-traffic window
- Container restart on renewal (~5s downtime every 60 days) — acceptable; no persistent connections except IMAP IDLE which clients auto-reconnect

## Rollback
Remove `-v /etc/letsencrypt:...` and `MAIL_TLS_*` env vars; server falls back to plaintext. Thunderbird users lose connection but app stays up.
