<audit_excerpts>
P0-01 (audit lines 30-49): Adicionar leads a uma campanha cria registros em `campaign_leads` com `nextScheduledAt = NULL`. O processor filtra `lte(campaignLeads.nextScheduledAt, now)`. Em SQL, `NULL <= now()` é `NULL` → tratado como `FALSE` → leads recém-adicionados nunca aparecem na query. Fix sugerido: adicionar `nextScheduledAt: new Date()` ao insert.

P0-09 (audit lines 247-281): `db.select(...)` retorna um query builder, não um valor escalar. Passar isso para `eq(column, subquery)` no Drizzle gera SQL inválido ou comparação serializada. Mesmo conceitualmente: pode haver múltiplas sequences por campanha. Pegar uma sequence aleatória sem ordering é não-determinístico. Fix sugerido: resolver em duas queries (firstSequence then firstStep), and return 400 if no step exists.

P0-10 (audit lines 283-298 + CONTEXT.md "P0-10 cascade fix uses both schema migration AND in-code transaction wrap"): Schema desenhado sem cascade. Fix sugerido: adicionar `onDelete: 'cascade'` no schema (preferível, com migration) OR fazer teardown manual nas rotas. CONTEXT.md decision: BOTH.
</audit_excerpts>
