GET /api/outreach/campaigns/:campaignId/sequences (from src/server/routes/outreach/campaigns.ts:568-604):
- No query params required.
- Response:
  ```typescript
  {
    sequences: Array<{
      id: string
      campaignId: string
      name: string
      description: string | null
      createdAt: string
      steps: Array<{
        id: string
        sequenceId: string
        stepOrder: number
        type: 'email' | 'delay' | 'condition'
        delayHours: number
        subject: string | null
        plainBody: string | null
        htmlBody: string | null
        abTestEnabled: boolean
        totalSent: number
        totalOpens: number
        totalClicks: number
        totalReplies: number
      }>
    }>
  }
  ```
- In v1, each campaign has exactly one sequence (the default "Main Sequence" created in POST /campaigns). For this plan, render `sequences[0]?.steps` and ignore additional sequences — if `sequences.length === 0`, show empty state.

GET /api/outreach/campaigns/:id/stats (from src/server/routes/outreach/campaigns.ts:1070-1121):
- No query params required (org membership derived from the campaign).
- Response:
  ```typescript
  {
    stats: {
      totalLeads: number       // count(*) of campaign_leads
      contacted: number        // count where status != 'new'
      replied: number          // count where status in ('replied','interested')
      bounced: number          // count where status = 'bounced'
      totalOpens: number       // sum of campaign_leads.totalOpens
      totalClicks: number      // sum of campaign_leads.totalClicks
      totalReplies: number     // sum of campaign_leads.totalReplies
    }
  }
  ```
- Compute rates client-side:
  - `openRate = totalOpens / max(contacted, 1) * 100`
  - `clickRate = totalClicks / max(contacted, 1) * 100`
  - `replyRate = replied / max(contacted, 1) * 100`
  - `bounceRate = bounced / max(contacted, 1) * 100`
- (Server's `/campaigns/stats` org-wide endpoint computes the same math — mirror it.)

Query cache reuse: BOTH tabs may call `GET /:campaignId/sequences`. Use the SAME queryKey `['campaign-sequences', organizationId, campaignId]` in both so React Query serves the second call from cache. Even though /stats doesn't need /sequences, the per-step breakdown table in StatsTab DOES.

Wouter routing: SequenceTab's Edit button uses `<Link href={\`/outreach/campaigns/${campaignId}/sequences/new\`}>` — this route already exists in main.tsx (registered in Phase 14, see src/main.tsx:450-456).
