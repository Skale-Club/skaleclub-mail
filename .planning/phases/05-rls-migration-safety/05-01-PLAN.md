# Phase 05 Plan 01: Fix RLS Policies (DBS-01)

## Objective

Fix all broken RLS policies that reference the removed `servers` table and `server_id` column. Replace server-scoped access checks with organization-scoped checks using `is_org_member()` / `is_org_admin()`.

**Purpose:** Data isolation between organizations is currently broken — policies reference columns and tables that no longer exist, so RLS either errors or is silently bypassed.
**Output:** New migration file `016_fix_rls_org_scoped.sql` + verification script.

## Context

@supabase/migrations/001_enable_rls.sql
@supabase/migrations/008_remove_server_legacy.sql
@supabase/migrations/004_outreach_rls.sql
@src/db/schema.ts

### Critical Problem

Migration `001_enable_rls.sql` defines:
- Helper functions `is_server_member`, `is_server_admin`, `is_server_editor` — query `public.servers` table (DROPPED in migration 008)
- RLS policies for `domains`, `credentials`, `routes`, `smtp_endpoints`, `http_endpoints`, `address_endpoints`, `messages`, `deliveries`, `track_domains`, `suppressions`, `statistics` — all use `server_id` column (does NOT exist in schema.ts; tables have `organizationId`)

Migration `008_remove_server_legacy.sql`:
- Dropped `is_server_member`, `is_server_admin`, `is_server_editor`
- Dropped `servers` table with CASCADE
- Recreated `is_webhook_member` with org-scoped access
- **Did NOT fix the RLS policies on the 11 server-scoped tables** — these policies still call the dropped functions

### Correct Pattern (from migration 004)

```sql
-- outreach tables use is_outreach_org_member(organization_id)
CREATE POLICY email_accounts_select ON public.email_accounts FOR SELECT
    TO authenticated
    USING (public.is_outreach_org_member(organization_id) OR public.is_platform_admin());
```

The existing `is_org_member(target_organization_id)` function (from migration 001, line 23) already provides org-scoped membership checks. The outreach tables created a redundant `is_outreach_org_member` that does the same thing.
